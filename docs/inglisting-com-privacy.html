Privacy Policy

Effective and last updated: July 13, 2026

This policy explains how ING Mining LLC ("ING Mining", "we", "us") handles information in connection with ING Listing Engine — this website (inglisting.com), the eBay account connection process, and the free desktop application. We built this tool to be as privacy-respecting as possible: almost everything runs on your own machine, and we designed the pieces that don't to hold as little data as possible, for as short a time as possible.

1. This Website

inglisting.com is a static informational site. It does not use tracking cookies, advertising pixels, or third-party analytics. Our hosting provider automatically logs standard web server data (IP address, browser user-agent, request timestamps) for security and abuse prevention, consistent with normal hosting operations. We do not use these logs for marketing or share them with third parties.

2. Connecting Your eBay Account

ING Listing Engine publishes listings on your behalf using eBay's official Sell API. To authorize this, you log in directly on eBay's own website — we never see or handle your eBay password.

After you approve access, eBay sends a one-time authorization code to our server (inglisting.com/api/ebay/callback/), which exchanges it for an access token and hands that token off to your local instance of the desktop app.

What this means in practice:

  • Your eBay access tokens are held on our server only long enough to be picked up by your desktop app — a maximum of 5 minutes.
  • Each token hand-off is single-use: it is deleted immediately once retrieved, or automatically expires if never claimed.
  • We do not permanently store your eBay tokens, listings data, sales data, or account information on our servers at any point.
  • The requested permissions (scopes) are limited to what's needed to manage listings: inventory, account, and order fulfillment access.

You can revoke ING Listing Engine's access to your eBay account at any time from your eBay account settings under Third-Party App Access.

3. The Desktop Application

Once connected, your eBay tokens and any API keys you provide (such as your own Anthropic Claude API key) are stored locally on your own computer, in a credentials.json file in the app's install folder. This file never leaves your machine and is not transmitted to ING Mining's servers. Product data you import (URLs, photos, generated listing content) is processed locally and sent only to the third-party services required to perform the action you requested — for example, eBay's API to publish a listing, or Anthropic's Claude API to generate listing text, using your own API key.

4. Third-Party Services

Depending on which features you use, the app communicates directly with:

  • eBay (api.ebay.com / auth.ebay.com) — to publish and manage listings, under eBay's own User Privacy Notice.
  • Anthropic (Claude API) — to generate titles, descriptions, and item specifics, using an API key you supply and control directly with Anthropic.
  • Stripe — only if you purchase an optional Pro license, to process payment. We do not receive or store your card details; Stripe handles that under its own privacy policy.

5. Data Retention & Security

Because we don't retain eBay tokens or personal listing data on our servers beyond the brief OAuth hand-off window described above, there is very little of yours for us to retain. Our relay endpoints run over HTTPS, and the temporary session files used during token hand-off are access-controlled by a single-use secret only your app instance receives.

6. Your Choices

  • Revoke eBay access anytime from your eBay account's Third-Party App Access settings.
  • Delete your locally stored credentials by removing the app's credentials.json file.
  • Uninstall the application at any time — no account or subscription is created on our end to cancel.

7. Children's Privacy

ING Listing Engine is a business tool intended for eBay sellers and is not directed at, or knowingly used to collect information from, children under 13.

8. Changes to This Policy

If this policy changes, we'll update the "last updated" date above. Continued use of the site or app after a change constitutes acceptance of the updated policy.
